📧 EmailJS Email Template Previews

Visual guide for setting up your Yaya Starchild email templates

📋 How to Use This Guide:
1. Copy the HTML code from each template section below
2. Paste into EmailJS Dashboard → Email Templates → Content tab
3. Configure the settings (From, To, Subject) as shown
4. Save and test each template

📧 Template 1: Newsletter Signup Notification

Purpose: Notifies you when someone subscribes to your newsletter

Sends to: faeriepoetics@gmail.com

Triggered by: Newsletter form submissions on homepage, shop, about page

Template Configuration

Template Name: Newsletter Signup Notification

From Name: Yaya Starchild Website

From Email: {{from_name}}

Reply To: {{reply_to}}

Send To: {{to_email}} ⚠️ Use variable, not hardcoded!

Subject: 🌟 New Newsletter Subscriber

Required Variables

{{to_email}} - Your email (faeriepoetics@gmail.com)

{{user_email}} - Subscriber's email address

{{page}} - Page they subscribed from (e.g., "Homepage")

{{timestamp}} - Date and time of subscription

{{from_name}} - Sender name (website)

{{reply_to}} - Email to reply to (subscriber's email)

🛍️ Template 2: Order Notification

Purpose: Notifies you when a customer places an order

Sends to: faeriepoetics@gmail.com

Triggered by: Successful Stripe payment completion

Template Configuration

Template Name: Order Notification

From Name: Yaya Starchild Website

From Email: {{from_name}}

Reply To: {{reply_to}}

Send To: {{to_email}} ⚠️ Use variable, not hardcoded!

Subject: 🛍️ New Order - ${{order_total}}

Required Variables

{{to_email}} - Your email (faeriepoetics@gmail.com)

{{customer_name}} - Customer's full name

{{customer_email}} - Customer's email address

{{order_total}} - Total order amount (e.g., "24.99")

{{order_items}} - List of items ordered

{{shipping_address}} - Full shipping address

{{timestamp}} - Date and time of order

{{from_name}} - Sender name (website)

{{reply_to}} - Customer's email for replies

⚠️ IMPORTANT REMINDER:

When setting up templates in EmailJS:
  • Always use {{to_email}} in the "Send To" field, NOT a hardcoded email
  • Grant ALL permissions when connecting Gmail
  • Test each template after creating it
  • Copy the HTML exactly as shown (including all style attributes)
